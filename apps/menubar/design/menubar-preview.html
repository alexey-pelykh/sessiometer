Sessiometer — menubar popover · all 9 states

The macOS status-bar popover across every launch-or-attach state, at true 380 px width, in the native macOS language — vibrancy, meters, drawn sparklines, label-primary identity. Each state is a distinct icon-shape + panel message + affordance; the menubar never renders healthy on a degraded daemon. A visual build-reference for the SwiftUI panel (#168/#169) — it does not port to code.

Design notes — the 9-state map in the native macOS language

  • One surface, nine states, one language. Every state is a render of the same 380 px vibrancy popover — healthy rosters and degraded messages share the frosted material, hairline dividers, SF-symbol-style glyphs and tabular numerals. Each state is a distinct icon-shape + panel message + affordance (the 9-state map); the caption glyph above each card is that state’s menubar shape.
  • Healthy = the status render. Status and Stats share one language: label-primary two-line identity (Work / Personal / Scratch + monogram badge, provider below), session & weekly usage meters on the status CLI’s own bands (green <75 / amber 75–89 / red ≥90) with resets in its compact duration form (2h14m, 3d), the smart swap hero callout, and drawn SVG sparklines with the neutral symmetric SIGNAL band. Every percent carries its band in colour — a healthy one included — exactly as the CLI paints it.
  • Never healthy on a degraded daemon (#137). Disconnected shows the last-known roster dimmed + “stale” with a broken-link badge and a Reconnect escape — never frozen-as-live; swaps paused. Crash-looping persists a fault shape and holds status until a fresh snapshot survives a stability window (the healthy-flash is debounced).
  • Freshness is a spectrum. Stale-snapshot is milder than disconnected — still connected, so the full roster renders live with only a subtle amber stale 6m modifier in the header and an amber freshness line in the footer; the swap stays enabled (the daemon re-mints viability on swap).
  • Blocked states are honest and non-intrusive. Keychain-locked keeps the fresh roster but marks the active account with an orange at-risk lock — the refresh path is broken while the token may still work, so it outranks a mere amber stale mark (🟡 < 🟠, the CLI’s ladder) — pauses the swap callout, and only waits: the app never prompts or auto-unlocks. Version-skew refuses to render any % and offers a copy-command instead.
  • Capture is a real action; copy-command only where the app can’t act. First-run / empty-roster onboarding captures the active account in-app (#360) — a button + optional label field that sends the verb over the #358 control socket and renders an honest pending → done → error from the redacted ack (no credential ever reaches the client). It is an onboarding affordance: a populated panel carries no capture bar, so the steady state stays a read-only glance and adding an account lives off-panel. The captured row then arrives on its own via the live watch stream — the affordance never inserts it. To add a different account the operator runs claude /login first, then captures (capture snapshots the active account; it is not a picker). Only where the app genuinely can’t act does it still copy a command: version-skew’s Copy update command (brew upgrade sessiometer). daemon-starting stays a static forming glyph — the app fakes no progress it isn’t doing.
  • Empty-roster is distinct from daemon-down. First-run uses a calm accent-tinted add mark and inviting copy — unmistakable from the broken-link disconnected shape or the fault triangle.
  • Provider-neutral, minimal identifiers, every state. Wherever accounts render, each provider gets the same monochrome monogram badge and a plain text label (no brand colour or logo). The operator-chosen label leads and the secondary line is provider-only; with no custom label an account's label defaults to its account UUID (the roster stores no email — it lives only in the encrypted credential). The footer swap-history now reads by alias (Work → Personal).
  • No-wrap by construction, every state. Aliases own their line and truncate; badges, tags, health glyphs, strip actions and swap buttons are fixed-width and can never be pushed off. Meters use a fixed 4-column grid so % and reset countdowns stay column-aligned.
  • Light + dark hold uniformly. Healthy is shown in both themes; degraded states in Light with Dark spot-checks (not-running, keychain-locked, empty-roster). Every degraded token — strips, fault/attention badges, the orange at-risk lock and the amber stale modifiers — is themed, so dark applies uniformly to all nine.
